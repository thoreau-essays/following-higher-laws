{%- comment -%} THEME FONTS ----------- Emits theme-specific font CDN links and CSS-variable overrides. Activation: Set in _data/theme.yml: base-font-family: theme (and optionally) display-font-family: theme When `base-font-family: theme`, each color-theme's recommended body face is loaded from Google Fonts and assigned to --cb-font-family-base. When `display-font-family: theme`, the matching heading face is loaded and assigned to --cb-font-family-display. Users can override either independently: base-font-family: "'Literata', Georgia, serif" display-font-family: "'Playfair Display', Georgia, serif" font-cdn: "https://fonts.googleapis.com/css2?family=Literata..." When base-font-family is set to a value other than 'theme' (and is not blank), the user's explicit font-cdn (if any) is loaded instead. All theme fonts are free, openly licensed, and chosen for long-form reading. Each has at minimum: regular, italic, and a 500-weight cut. {%- endcomment -%} {%- assign base_family = site.data.theme.base-font-family -%} {%- assign display_family = site.data.theme.display-font-family -%} {%- assign theme_name = site.data.theme.color-theme | default: 'default' -%} {%- comment -%} Should we resolve EITHER family from the theme?
